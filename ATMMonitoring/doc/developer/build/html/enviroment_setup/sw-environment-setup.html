Software Environment Setup¶

This page explain how to install the following tools:

  • Java Runtime Environment / Java Development Kit: The Runtime environment is needed to run java applications while the Developer Kit is for compiling java applications.
  • Apache Tomcat: Servlet container where the application will run. A Servlet container is needed in order to process JSP pages and run Java Web Applications.
  • Apache Maven: Is a software project management and comprehension tool, is used to run tests, compile, package and deploy the application.
  • Jenkins: Continuous integration software used to automatize the task of test, compiling and package versions of the application and deploy.
  • Javasphinx: Sphinx plugin that reads the javadoc in the source files and generate rst files.
  • GIT: Revision control tool
  • Eclipse: Integrated development environment (IDE).

The tools will be divided in:

  • Development Environment (only for development onto the application and testing)
  • Production Environment (does not need the previous tools, with the sole exception of the JRE/JDK )

Development Environment¶

Java Runtime Environment / Java Development Kit¶

CentOs by default has a JRE (Java Runtime Environment) installed, but we recommend installing a JDK (Java Development Kit), mainly for development purposes. In order to do so, follow the next steps:

  1. Go to Oracle Java downloads

  2. Select the JDK

  3. In the page Java SE Development Kit 7 Downloads select the latest JDK package according to your architecture and OS:
    • jdk-7u51-linux-x64.rpm for 64 bit Linux.
    • jdk-7u51-linux-i586.rpm for 32 bit Linux, or if you are unsure of your architecture.

  4. After the file has been downloaded, open a console and write the following command:

    $ rpm -Uvh jdk-7u45-linux-<i586|x64>.rpm

Verifying the JDK Installation¶

Execute in a command line:

$ javac -version

Eclipse¶

  1. Download Eclipse from the official site
  2. Unzip/Untar the file in a user folder (e.g. /home/user/)

Verifying the Eclipse installation¶

  1. Go to the Eclipse installation folder (e.g. /home/user/eclipse)
  2. Execute the file called eclipse (with no extension)

Git¶

  1. Execute in the command line:

    $ yum install git

Verifying the Git Installation¶

  1. Execute in the command line

    $ git --version

Javasphinx¶

In order to install javasphix it is necessary to install sphinx 1.2 first

Sphinx Installation¶

  1. Open a console and execute the following command:

    $ easy_install -U Sphinx
Verifying the Sphinx Installation¶

  1. In a command line execute:

    $ sphinx-build

    and verify that the installed version is 1.2+

Javasphinx Installation¶

  1. Verify that the installed version of BeautifulSoup is not 4 through the Python interpreter with the following console commands:

    $ python
>>> import BeautifulSoup

    Note

    if this throws an error it means that BeautifulSoup isn’t currently installed.

    >>> print BeautifulSoup.__version__

    Note

    in order to exit the Python interpreter, just write:

    >>> quit()

    If the version 4 is present, we must uninstall it:

    $ easy_install -m BeautifulSoup4

  2. Install BeautifulSoup 3.2.1 with the next command:

    $ easy_install -U "BeautifulSoup==3.2.1"

  3. After installing BeautifulSoup 3.2.1, execute the following command to install javasphinx:

    $ easy_install -U "javasphinx==0.9.8"
Verifying the Javasphinx Installation¶

  1. Execute the following command:

    $ javasphinx-apidoc
Using Javasphinx¶

  1. In a command line write the following:

    $ javasphinx-apidoc  -o <output path where the rst files will be written> <path to the .java files folder>

  2. Execute the sphinx quickstart command:

    $ sphinx-quickstart
    • You will be presented a few questions like project name, author and version.
    • Select that you want to separate the build folder from the source folder.
    • After specifying all the requested parameters, two folders will be available, the build and the sources folder.

  3. Move all the files generated by the javasphinx-apidoc command to the sources folder, that was generated using sphinx-quickstart

  4. Rename the file packages.rst to index.rst , or copy its contents into index.rst

  5. Edit the sources/config.py and in the extension list, add ‘javasphinx’

  6. Execute in the root folder (where the sources and build folders are):

    $ make html

    Note

    the HTML documentation will be in the build folder

Production Environment¶

Apache Tomcat¶

  1. Download the binary files from the Eclipse official site.
  2. Unzip the file wherever you want (we recommend doing it in /var/lib/, and that’s what we’ll assume from now on).

Configuration For Deploying With Maven¶

  1. Edit the file conf/tomcat-users.xml adding the following:
<tomcat-users>
        <role rolename="manager-gui"/>
        <role rolename="manager-script"/>
        <role rolename="manager-jmx"/>
        <role rolename="manager-status"/>
        <role rolename="admin-gui"/>
        <role rolename="admin-script"/>
        <user username="admin" password="tomcat" roles="manager-gui,manager-script,manager-jmx,manager-status,admin-gui,admin-script"/>
</tomcat-users>

Running Tomcat as a Service¶

  1. Go to /etc/init.d and create a file called tomcat (with no extension):
    $ cd /etc/init.d

    $ vi tomcat

  2. Copy the following and paste it inside the file:

    #!/bin/bash
# description: Tomcat Start Stop Restart
# processname: tomcat
# chkconfig: 234 20 80
JAVA_HOME=/usr/java/jdk1.7.0_45
export JAVA_HOME  PATH=$JAVA_HOME/bin:$PATH
export PATH  CATALINA_HOME=/var/lib/apache-tomcat-7.0.47
case $1 in  start)
        sh $CATALINA_HOME/bin/startup.sh;;  stop)
        sh $CATALINA_HOME/bin/shutdown.sh;; restart)
        sh $CATALINA_HOME/bin/shutdown.sh
        sh $CATALINA_HOME/bin/startup.sh;;  esac
exit 0
  3. Change the permissions to the tomcat file:
    $ chmod 755 tomcat
  4. Execute:
    $ config --add tomcat

    $ chkconfig --level 234 tomcat on
  5. Verify the execution:
    $ chkconfig --list tomcat
  6. Execute the following to start the service:
    $ service tomcat start

Note

you may need to change file /var/lib/apache-tomcat-7.0.47/bin/catalina.sh permissions. For doing so, execute the following:

$ chmod 755 /var/lib/apache-tomcat-7.0.47/bin/catalina.sh

Verifying the Tomcat Installation¶

  1. Open a browser and go to http://localhost:8080, if everything was correct you will see the following:

Apache Maven¶

  1. Download the binary files from the Apache official site.

  2. Unzip the file wherever you want (we recommend doing it in /usr/local, and that’s what we’ll assume from now on).

  3. Rename the recently created folder to maven.

  4. Create the file etc/profile.d/maven.sh:
    $ vi etc/profile.d/maven.sh

  5. Add the following:

    export M2_HOME=/usr/local/maven
export PATH=${M2_HOME}/bin:${PATH}

  6. Save the file.

Test the Installation¶

  1. Logout and login again.

  2. Execute in a command line:
    $ mvn -version

Configure Maven to Deploy in Tomcat¶

  1. Update the file /usr/local/maven/conf/settings.xml by adding the following:
<server>
        <id>TomcatServer</id>
        <username>admin</username>
        <password>tomcat</password>
</server>

Note

the username and password must match the ones defined in tomcat-users.xml

Jenkins¶

  1. Execute the following
    $ sudo wget -O /etc/yum.repos.d/jenkins.repo http://pkg.jenkins-ci.org/redhat/jenkins.repo
$ sudo rpm --import http://pkg.jenkins-ci.org/redhat/jenkins-ci.org.key
$ yum install jenkins

Optional Configuration¶

  1. Edit the file /etc/sysconfig/jenkins changing the following values
    • JENKINS_PORT=”8080” by JENKINS_PORT=”8081”
    • add JENKINS_AJP_PORT=”8012”

Note

this way we avoid a port conflict if an instance of Tomcat (the default port is 8080) is running in the same server

Verifying the Jenkins Installation¶

  1. Execute service jenkins start
  1. In a browser, go to http:localhost:8081 and check that you can see the following:

Adding the Git Plugin¶

  1. To install plugins it is necessary to select the option Manage Jenkins available in the left menu:

  2. Select Manage Plugins:

  3. Select the Available tab:

  4. From the list select git plugin.

JRE/JDK Configuration¶

  1. In the main page select Manage Jenkins:

  2. Select the option Configure System:

  3. Now search for the option JRE/JDK installation:

  4. Uncheck the option Install Automatically:

  5. In the option JAVA_HOME write the path to your local JRE/JDK installation. For example:
    • /usr/java/jdk1.7.0_45

    Note

    the path being used in this example corresponds to the installation path of the JDK described in previous sections, if the JDK was installed in another path, please put the one that corresponds to your installation

  6. Save the changes.

Maven Configuration¶

  1. Repeat the steps 1 to 4 of the previous section (JDK configuration), changing the 3rd step for the Maven Option:

  2. In the MAVEN_HOME option, write: * /usr/local/maven

    Note

    the path being used in this example corresponds to the installation path of Maven described in previous sections, if Maven was installed in another path, please put the one that corresponds to your installation.

  3. Save the changes.
